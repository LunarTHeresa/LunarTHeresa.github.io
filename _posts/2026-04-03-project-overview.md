## 为什么做这个题目？

期末Java课的项目选题，大家第一反应往往是：
- ❌ 学生管理系统
- ❌ 图书馆管理系统
- ❌ 银行账户系统

这些题目做出来像是在完成作业，没有灵魂。

我们希望做一个**真正能玩、真正有趣**的项目——一个TCG卡牌对战游戏。

---

## 游戏玩法

类似《炉石传说》的简化版，核心规则如下：

### 基础设定
- **双方英雄**：各30点生命值，降到0即失败
- **能量系统**：初始1点，每回合+1，上限10点
- **战场**：5个随从格子，双方各占一边
- **手牌**：最多10张，出牌消耗对应能量

### 回合流程

```
┌──────────────────────────────────────────────┐
│              我的回合                         │
├──────────────────────────────────────────────┤
│ ① 回合开始 → 抽1张牌，能量重置+1上限           │
│ ② 行动阶段 → 出牌 / 施放技能 / 跳过            │
│ ③ 战斗阶段 → 随从按顺序攻击                    │
│ ④ 回合结束 → 触发亡语，切换对手回合             │
└──────────────────────────────────────────────┘
```

### 卡牌类型

| 类型 | 说明 | 示例 |
|------|------|------|
| 随从牌 | 召唤到战场，可攻击 | 火焰剑士、龙息 |
| 法术牌 | 立即生效，消耗后消失 | 火焰冲击、龙息 |
| 英雄牌 | 暂不实现（简化版） | — |

## 项目目标

通过这个项目，我们希望展示：

1. **扎实的OOP功底** — 用设计模式而非if-else解决问题
2. **完整的工程能力** — 前后端分离、模块化、版本控制
3. **真实的团队协作** — 6人分工、代码规范、PR合并
4. **可玩的最终产品** — 不是半成品，是能打赢AI的完整游戏
